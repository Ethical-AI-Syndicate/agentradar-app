AgentRadar API Documentation

Real Estate Intelligence Platform API v1.0

API Status: Operational

API Overview

The AgentRadar API provides access to comprehensive real estate intelligence including property alerts, court filings, estate sales, and market analysis. Our API enables developers to integrate powerful real estate data and AI-driven insights into their applications.

Base URL

https://api.agentradar.app/

Current Version

All API requests are currently served from version 1.0. The API is RESTful and returns JSON responses.

Key Features

  • Real-time property opportunity alerts
  • Ontario court filing monitoring
  • Estate sale and probate tracking
  • AI-powered property analysis
  • Market intelligence and trends
  • Webhook notifications
  • RESTful API design

Authentication

AgentRadar API uses Bearer token authentication. Include your API token in the Authorization header of all requests.

Authorization: Bearer YOUR_API_TOKEN

Getting Your API Token

API tokens can be generated from your AgentRadar dashboard under Settings > API Access.

POST /api/auth/login

Authenticate and receive an access token

Core API Endpoints

GET /health

Check API health and status

{
  "status": "healthy",
  "timestamp": "2025-09-09T21:41:14.819Z",
  "uptime": 10.42,
  "environment": "production",
  "version": "1.0.0"
}
GET /api/alerts

Retrieve property alerts based on your preferences

Parameter Type Required Description
limit integer No Maximum number of alerts to return (default: 20)
page integer No Page number for pagination (default: 1)
type string No Filter by alert type (power_of_sale, estate_sale, etc.)
city string No Filter by city name
GET /api/court-filings

Access Ontario court filings and legal proceedings

GET /api/estate-sales

Retrieve estate sale and probate opportunities

POST /api/property-analysis

Analyze a property for investment potential and market value

Error Handling

The API returns standard HTTP status codes and includes detailed error information in JSON format.

Common Status Codes

  • 200 - Success
  • 400 - Bad Request
  • 401 - Unauthorized
  • 403 - Forbidden
  • 404 - Not Found
  • 429 - Too Many Requests
  • 500 - Internal Server Error

Error Response Format

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request parameters",
    "details": "The 'city' parameter must be a valid string"
  },
  "timestamp": "2025-09-09T21:41:14.819Z",
  "path": "/api/alerts",
  "method": "GET"
}

Code Examples

JavaScript/Node.js

// Fetch property alerts
const response = await fetch('https://api.agentradar.app/api/alerts', {
  headers: {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
  }
});

const alerts = await response.json();
console.log(`Found ${alerts.length} property alerts`);

Python

import requests

headers = {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.agentradar.app/api/alerts', headers=headers)
alerts = response.json()
print(f"Found {len(alerts)} property alerts")

cURL

curl -X GET "https://api.agentradar.app/api/alerts" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json"
